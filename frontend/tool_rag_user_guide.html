 Advanced — Chunker

 Search Scope
Top K 10 Range 1–50. Number of chunks fetched from the Vector Search index per query. Higher K improves recall but increases latency and LLM context cost. 8–15 is typical. Re-ranker then trims to Top N.
Query Type hybrid hybrid BM25 sparse + dense cosine (recommended) · similarity dense cosine only Hybrid wins on keyword-heavy and mixed queries. Similarity-only is faster but misses exact-match cases.
 Retrieval Enhancements
HyDE — Hypothetical Document Expansion ON Uses an LLM to rewrite the user query as a hypothetical answer document, then embeds that for retrieval. Significantly improves embedding alignment for vague or short queries. Adds ~1 LLM call latency per query.
Cross-Encoder Re-Ranking ON Passes each query+chunk pair through a cross-encoder model to reorder by true relevance. Usually the single highest-value quality lever. Reduces hallucination by surfacing the best chunks. Small latency cost.
 Advanced — Result Shaping
Re-rank Top N ADV 5 Range 1–20. After re-ranking, only the top N chunks are passed to the LLM as context. Lower N = tighter, cheaper context. Higher N = more recall but larger prompt and higher cost. Keep ≤ Top K.
Columns to Retrieve ADV chunk_id,source,page,content Comma-separated Delta table columns returned per chunk. Must include content. Adding metadata columns (e.g. author) enables richer citations. Removing content breaks synthesis.
 Advanced — HyDE
HyDE LLM Model ADV databricks-meta-llama-3-3-70b-instruct Any chat model accessible via your Databricks serving endpoint or external API. Larger models produce better hypothetical docs. Smaller models are faster. Match to your latency budget.
Apply HyDE On ADV retry retry only on CRAG re-query iterations · always every query · low_confidence when retrieval confidence is low retry is the best balance — saves latency on first-pass queries while rescuing weak ones.

 Agentic Pipeline — 3-Stage Orchestration NEW
Query Classifier NEW Automatic Runs automatically before any retrieval. An LLM decides: direct = answer from general knowledge, no retrieval needed (greetings, simple conversational turns) · retrieve = factual or domain-specific query, proceed to pipeline. Defaults to retrieve on any failure. Point 1 — Self-Directed Retrieval. Conversational queries no longer hit the vector index. Response shows decision: direct_answer. Zero retrieval cost and latency for simple turns. No configuration needed.
Query Planner NEW Automatic Runs after classification, before retrieval. An LLM decides if the query is simple (1 hop) or complex (multi-hop). Simple queries pass through unchanged. Complex queries are decomposed into 2–3 atomic sub-questions, each processed through the full CRAG loop independently. Results are accumulated and synthesized into one unified response. Point 3 — Goal-Persistent Multi-Step Reasoning. Multi-hop queries (comparisons, aggregations, multi-document summaries) get separate retrieval passes per sub-question. Response shows decision: multi_step with sub_questions and sub_answers fields in the trace. No configuration needed.
Pipeline Decision Values NEW — Every CRAG response includes a decision field: direct_answer classifier routed to general LLM, no retrieval · multi_step query decomposed and answered via multiple retrieval passes · pass retrieval scored above threshold, generated normally · requery low score triggered re-query loop · web_fallback retries exhausted, answered from web search · fail all sources failed, best-effort response returned Use the decision value in the response tag (visible in the chat UI after every RAG reply) to understand which path was taken. Useful for debugging — if you see requery frequently, your chunks may be too large or your threshold too high.
 Grading & Retry — Self-Correction on Failure (Point 2)
Relevance Threshold 0.60 Range 0–1. Minimum grade score for retrieved chunks to be considered acceptable. Below this → re-query. Too high = excessive retries. Too low = poor chunks pass through. 0.55–0.65 works well. Must be > Web Fallback Trigger.
Max Re-Query Iterations 2 Range 0–5. Hard cap on CRAG loop attempts. Recommended ≤ 2. Each iteration adds full retrieval + grading latency. Values > 2 rarely improve quality and spike response time.
 Web Fallback
Web Fallback Search ON When all CRAG retries still score below Relevance Threshold, fall back to live web search. Ensures the pipeline never returns a null answer. Disable for air-gapped or strictly controlled knowledge bases.
Web Search Provider tavily tavily best results, requires TAVILY_API_KEY · serpapi Google results, requires SERPAPI_KEY · duckduckgo no key needed, lower quality Tavily is purpose-built for RAG and returns clean, structured web excerpts. DuckDuckGo is free but noisier.
 Advanced — Relevance Grader
Grader LLM Model ADV gemini-2.0-flash LLM used to score each chunk 0.0–1.0 for relevance. Smaller/faster models work well here. Supported: gemini-2.0-flash (default, Gemini API) · databricks-meta-llama-3-3-70b-instruct (Databricks fallback). GPT models are not supported — the pipeline will fall back to Databricks automatically. Used for all 3 agentic LLM calls (classifier, planner, grader). Gemini is preferred — faster and lower cost. Databricks is the automatic fallback when no Gemini key is present. Does not need to match the generation LLM.
Docs Graded per Run ADV 3 Range 1–10. How many of the Top K chunks are submitted to the grader LLM per iteration. Grading all K chunks is expensive. 3–5 gives a reliable signal. The top chunks (by VS score) are always picked first.
Grader Temperature ADV 0.00 Range 0–1. Temperature for the grader LLM. Keep at 0 for deterministic yes/no relevance verdicts. Any value above 0 introduces noise into grading decisions. Always keep at 0.
 Advanced — Re-query Loop
Re-query Strategy ADV hyde hyde HyDE expansion (recommended) · decompose break into sub-questions · rephrase simple paraphrase HyDE is most effective for retrieval improvement. Decompose is better for multi-part questions.
Max Web Results ADV 5 Range 1–20. Number of web search results retrieved during fallback. More results = wider coverage but noisier context. 3–7 is ideal. Increase for highly specific or niche queries.
Web Fallback Trigger Score ADV 0.40 Range 0–1. Grade score below which web search is triggered. Must be < Relevance Threshold. Creates a "retry window" between trigger and threshold. If 0.40 trigger and 0.60 threshold, scores 0.40–0.60 trigger re-query, below 0.40 goes straight to web.

 Answer Quality Gate
NLI Faithfulness Check ON Runs DeBERTa-v3 NLI entailment check: does the answer logically follow from the retrieved context? Primary hallucination guard. Unfaithful answers are discarded and regenerated. Keep ON in production.
Faithfulness Threshold 0.70 Range 0–1. NLI entailment score below which the answer is rejected and regeneration is triggered. Lower threshold = more answers pass but more hallucinations slip through. Higher = stricter but more regenerations and latency.
 Advanced — LLM
Max Output Tokens ADV 2048 Range 256–16,384. Maximum tokens the LLM can generate per answer. Too low = truncated answers. Too high = unnecessary cost. 1,024–2,048 covers most Q&A use cases.
System Prompt ADV Expert assistant with citation instructions Free-text system prompt injected before synthesis. Must instruct citation format [SOURCE: chunk_id] if citations are enabled. Most impactful single-field tuning lever. Tone, format, verbosity, and citation style all stem from here.
 Advanced — Citations
Citation Injection ADV ON Instructs the LLM to append [SOURCE: chunk_id] inline for every factual claim. Enables source traceability. Required by the Pre-Production Checklist. Keep ON for all production deployments.
Citation ID Resolution Check ADV ON Post-generation: validates that every cited chunk_id actually exists in the Delta table. Catches fabricated citations. Fabricated IDs are flagged in the UI with a warning badge.
 Advanced — NLI Model

 Tracing
MLflow Tracing ON Instruments every pipeline node with MLflow traces: latency, relevance scores, retrieval counts. Essential for production monitoring and debugging. Very low overhead. Keep ON always.
 Evaluation
Ragas Evaluation ON Enables the automated Ragas evaluation panel. Does not run automatically — triggered manually in the Test panel. Enables quality scoring against a test set. Required by the Pre-Production Checklist.
Metric: Faithfulness ON Measures whether the answer is grounded in the retrieved context (not hallucinated). Most critical metric. Low faithfulness = hallucination problem. Target ≥ 0.70.
Metric: Answer Relevancy ON Measures whether the answer directly addresses the question asked. Low relevancy = LLM drifting off-topic or system prompt too loose. Target ≥ 0.75.
Metric: Context Recall ON Requires ground truth. Measures how much of the expected answer is covered by retrieved chunks. Low recall = retrieval missing key documents. Increase Top K or improve embedding model.
Metric: Context Precision ON Requires ground truth. Measures the proportion of retrieved chunks that are actually relevant. Low precision = too much noise in context. Enable re-ranker and lower Top N.
